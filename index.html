⚙️ Groovy Script Test Step: Deep Dive

A Groovy Script runs as a standalone test step in the execution sequence. It has full access to the testRunner object, allowing it to manipulate other steps, adjust endpoints, and dictate the test execution flow. Below is a breakdown of an automated calculation flow using the public Calculator WSDL (Add operation).

🔗

Real-world endpoint: The code snippets reference the “Add” operation from http://www.dneonline.com/calculator.asmx?wsdl — the SOAP request expects intA and intB and returns AddResult.

📦 1. Imports & Setup

import com.eviware.soapui.support.XmlHolder
import com.eviware.soapui.impl.wsdl.testcase.WsdlTestRunContext
  • XmlHolder enables XPath‑based reading and dynamic modification of XML documents. Essential for manipulating SOAP request/response envelopes.
  • WsdlTestRunContext provides the necessary runtime environment to trigger a test step programmatically via testStep.run().

🏷️ 2. Working with TestSuite Properties

log.info context.expand('${#TestSuite#Temp}')
log.info testRunner.testCase.testSuite.project.testSuites["TestSuite"].getPropertyValue("Temp")
testRunner.testCase.testSuite.project.testSuites["TestSuite"].setPropertyValue("Temp","50")
  • context.expand() seamlessly resolves property expansions at runtime, making it highly flexible.
  • setPropertyValue updates properties on-the-fly, persisting the values throughout the remainder of the test execution cycle.

📄 3. Reading & Modifying the Request XML (Calculator Add Example)

def addReq = testRunner.testCase.testSuite.testCases["Add TestCase"].testSteps["Add"].getPropertyValue("RawRequest")
def xmlAdd = new XmlHolder(addReq)
xmlAdd.setNodeValue("//*:intA","15")
xmlAdd.setNodeValue("//*:intB","15")
def newxml = xmlAdd.getXml()
testRunner.testCase.testSuite.testCases["Add TestCase"].testSteps["Add"].setPropertyValue("Request", newxml)
  • RawRequest securely holds the template of the last sent request.
  • The wildcard //*:intA bypasses strict namespace bindings — perfect for the calculator’s XML structure.
  • Overwriting the step’s Request property dynamically alters the payload that will be fired when the step is invoked.

⚙️ 4. Programmatic Execution & Assertion (AddResult validation)

// Run the Add step from the public calculator service
def addteststep = testRunner.testCase.testSuite.testCases["Add TestCase"].testSteps["Add"]
def contextadd = new WsdlTestRunContext(addteststep)
addteststep.run(testRunner, contextadd)

// Extract AddResult from response and assert
def addResponse = addteststep.getPropertyValue("Response")
def xmlAddresonse = new XmlHolder(addResponse)
def sum = xmlAddresonse.getNodeValue("//*:AddResult")
assert sum == "30"   // 15+15 = 30

🧪 Script Assertions

A Script Assertion attaches directly to an HTTP request test step (SOAP, REST, JDBC). It fires immediately after the response is received. It relies on three built‑in variables:

  • log – Logger interface targeting the internal test execution log.
  • context – Provides the TestCaseRunContext to read/write properties.
  • messageExchange – Grants direct access to raw request/response objects, HTTP headers, latencies, and file attachments.
⚠️

Note: The testRunner object is not available inside Script Assertions. You must rely on context.getTestCase() to interact with the parent suite hierarchy.

➕ Calculator Service Assertion (Add operation)

import com.eviware.soapui.support.XmlHolder

// Validate AddResult from public calculator SOAP response
def holder = new XmlHolder(messageExchange.responseContentAsXml)
def addResult = holder.getNodeValue("//*:AddResult")

def expectedSum = "30"   // based on sent intA=15, intB=15
assert addResult == expectedSum : "Addition mismatch: expected ${expectedSum}, got ${addResult}"
log.info "✅ AddResult validation passed: " + addResult

📄 XML Payload Validation (With Namespaces)

import com.eviware.soapui.support.XmlHolder

def holder = new XmlHolder(messageExchange.responseContentAsXml)
holder.declareNamespace("ns1", "https://maps.googleapis.com/maps/api/place/textsearch")

def actual = holder.getNodeValue("/ns1:Response[1]/ns1:results[1]/ns1:e[1]/ns1:reference[1]")
def expected = context.getTestCase().getPropertyValue("ID")

assert actual == expected

📦 JSON Status Validation & Data Extraction

import groovy.json.JsonSlurper

def response = messageExchange.response.responseContent
def jsonParser = new JsonSlurper().parseText(response)

assert jsonParser.status == "OK"
assert jsonParser.name == "Backline house"

// Storing extracted data for downstream steps
context.getTestCase().setPropertyValue("place_id", jsonParser.place_id.toString())

🌐 Comprehensive API Testing Strategies

To scale your automation efforts into a robust, enterprise-grade architecture, utilize SoapUI's broader capabilities outside of purely functional scripts. The public calculator WSDL (http://www.dneonline.com/calculator.asmx?wsdl) is perfect for practicing these techniques:

  • Data-Driven Testing (DDT): Avoid hardcoded values. Utilize Groovy coupled with Apache POI to read bulk Excel sheets or CSVs, iterating rows to blanket the API with boundary and negative test cases.
  • Service Virtualization (Mocking): Don't wait for backend completion. Spin up local Mock Services in SoapUI. Configure dispatch styles (Sequence, Random, or Groovy Scripted) to emulate complex downstream JSON/XML responses based on incoming request headers.
  • Security Vulnerability Scanning: Attach automated Security Tests over your functional endpoints to continuously scan for SQL Injections, Cross-Site Scripting (XSS), and XML Bomb/Fuzzing attacks.
  • Performance & Load Simulation: Instantly right-click a functional test to generate a Load Test. Apply strategies like Burst, Variance, or Simple threads to measure Transactions Per Second (TPS) and latency degradation.
  • CI/CD & CLI Automation: Integrate your XML project files into Jenkins or GitLab pipelines using the native CLI runner. Output JUnit format reports to fail builds automatically if regression assertions drop. 
    testrunner.bat -s"TestSuite 1" -r -a -j -f"C:\Reports" "C:\Project.xml"

🎯 Architecture Best Practices

  • Validate Network Responses First: Always verify that messageExchange.response.responseContent is not null before instantiating JSON/XML parsers to avoid NullPointer exceptions on server timeouts.
  • Strict Namespaces: Relying purely on wildcards (//*:node) is fragile for deeply nested XML. Always use declareNamespace() for robust, deterministic XPath targeting.
  • Single Responsibility Assertions: Keep script assertions focused. One logical check per assertion block yields hyper-specific logs when a failure occurs.
  • Graceful Exception Handling: Wrap complex serialization logic inside try/catch blocks, triggering context.getTestCase().fail("Message") to fail the test gracefully without crashing the suite executor.
  • Environment Management: Define endpoints using Project-level Custom Properties (e.g., ${#Project#BaseUrl}). This permits instant toggling between Dev, QA, and UAT servers without altering a single script.

📜 Complete Master Script

Below is the fully assembled Groovy Script ready for copy-pasting. This script is designed for the public Calculator SOAP service available at http://www.dneonline.com/calculator.asmx?wsdl (Add operation). Ensure you have imported this WSDL into SoapUI and have an existing test case named Add TestCase containing a SOAP step named Add.

import com.eviware.soapui.support.XmlHolder
import com.eviware.soapui.impl.wsdl.testcase.WsdlTestRunContext

log.info '--- Groovy Script Execution Started (Calculator Service) ---'

// 1. Property Management
def project = testRunner.testCase.testSuite.project
project.testSuites["TestSuite"].setPropertyValue("Temp", "50")
log.info "Set Property Value: " + context.expand('${#TestSuite#Temp}')

// 2. Dynamic Request Modification (intA & intB for Add operation)
def targetStep = testRunner.testCase.testSuite.testCases["Add TestCase"].testSteps["Add"]
def rawReq = targetStep.getPropertyValue("RawRequest")

if(rawReq) {
    def xmlAdd = new XmlHolder(rawReq) 
    xmlAdd.setNodeValue("//*:intA", "15")
    xmlAdd.setNodeValue("//*:intB", "15")
    targetStep.setPropertyValue("Request", xmlAdd.getXml())
} else {
    log.error "RawRequest was empty!"
}

// 3. Programmatic Execution
def runContext = new WsdlTestRunContext(targetStep)
def result = targetStep.run(testRunner, runContext)
log.info "Step execution status: " + result.status

// 4. Result Extraction & Assertion (AddResult validation)
def responseXml = targetStep.getPropertyValue("Response")
if(responseXml) {
    def xmlParser = new XmlHolder(responseXml) 
    def sum = xmlParser.getNodeValue("//*:AddResult")
    
    log.info "Calculated Sum: " + sum
    assert sum == "30" : "Math validation failed! Expected 30, got ${sum}"
} else {
    testRunner.fail("No response payload received from calculator service.")
}
